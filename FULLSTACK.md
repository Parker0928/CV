### 🤖 AI Agent Platform（智能数据助手平台）

> 🕐 **2025.09 – 至今** &nbsp;&nbsp;|&nbsp;&nbsp; 👤 全栈技术负责人

基于 LLM + RAG + AI Agent 构建的智能数据助手平台，用户可通过自然语言查询链上数据、数据集信息及市场分析结果；系统通过向量检索结合大模型推理生成结构化数据，实现智能数据分析与知识问答能力。

**🔩 技术栈：** `Monorepo` `Vue3` `TypeScript` `NestJS` `OpenAI API` `LangChain` `RAG` `Embedding` `SSE Streaming` `Vite` `PostgreSQL`

**前端层：**

- 构建 **AI Chat 前端应用**，基于 **Vue3 + Pinia** 实现多轮对话状态管理，设计 `conversation → message → chunk` 三级数据模型，支持**上下文窗口管理**（Token 裁剪策略）、历史会话持久化及流式消息逐字渲染，支撑 **100+ 并发用户**实时 AI 交互。  
- 利用 **Server-Sent Events**（`text/event-stream`）实现 LLM 推理结果**流式返回**，前端基于自定义 `useSSE` 组合式函数处理流解析、错误重连与 Token 计数，首字响应时间**控制在 200ms 以内**，用户体感生成速度**提升约 3 倍**。  
- 基于 **Prompt Engineering** 设计结构化 Prompt 模板体系，通过 System Prompt 约束输出格式（JSON Schema / Markdown Table），引导模型稳定输出结构化数据，前端结合 ECharts 动态渲染图表，图表生成准确率达 **95%+**。

**后端层：**

- 服务端基于 **NestJS 模块化架构**构建 AI Service 层，将 **LLM Module、RAG Module 与 Agent Module** 职责分离，通过依赖注入实现模块解耦；封装统一的 `LLMProvider` 抽象层，支持**多模型（GPT-4o / Claude / DeepSeek）热切换与降级策略**（熔断器模式 + 指数退避重试）。  
- 基于 **RAG（Retrieval Augmented Generation）** 架构对数据集文档与链上数据说明进行 **Embedding 向量化处理**，采用文本分块（Chunking）+ 重叠窗口策略优化切片质量，通过向量数据库进行语义检索并注入 Prompt 上下文，检索准确率**从 65% 提升至 92%**。  
- 构建 **AI Tool / Function Calling** 机制，定义标准化工具描述协议（Tool Schema），实现链上数据查询、数据集信息检索、行情 K 线分析等 **10+ 种工具能力**，支持工具链式调用与并行执行。  
- 设计会话与消息持久化方案（PostgreSQL），实现历史对话存储、Token 用量统计与用户配额管理。

**工程层：**

- 使用 `pnpm workspace` + **Monorepo 架构**搭建全栈工程，划分 `apps/web`、`apps/server`、`packages/sdk`、`packages/ui` 四层目录结构，实现跨应用**代码复用率 60%+**，依赖版本统一管理。

### 📊 AI DATA（链上数据中心）

> 🕐 **2024.11 – 至今** &nbsp;&nbsp;|&nbsp;&nbsp; 👤 全栈技术负责人

开放人工智能的链上数据中心平台，包含四大核心产品线：**Data Market**（数据集上架、检索与交易市场）、**Data Pump**（数据集资产化铸造与发行）、**Data Works**（数据标注与多模型评测工作台）、**Points**（用户积分激励体系）。

**🔩 技术栈：** `Monorepo` `Vue3` `TypeScript` `Pinia` `Tailwind CSS` `NestJS` `PostgreSQL` `Redis` `Privy` `genapi` `Konva` `WebSocket` `TradingView`

**前端层：**

- 基于 **WebSocket 长连接**实现实时行情数据推送，封装自动重连、心跳保活与消息队列缓冲机制，数据延迟**控制在 50ms 以内**；集成 **TradingView Charting Library** 实现专业级 K 线与深度图，通过 `Datafeed API` 对接自有行情源，支撑**万级 QPS** 数据渲染。  
- 设计并实现**大文件分片上传引擎**，前端通过 **Web Workers** 进行文件切片（默认 5MB/片）与 SparkMD5 Hash 计算，利用 Axios 并发上传队列（最大并发 6 路），支持 **TB 级数据集上传**、断点续传（成功率 **99.9%**）与上传进度实时展示，单文件最大支持 **100GB**。

- 集成 **Privy** 实现 **Web3 身份认证**，支持钱包登录（MetaMask / WalletConnect）与社交账号（Google / Twitter）**统一登录**，降低用户 Web3 准入门槛。

**后端层：**

- 基于 **NestJS** 搭建数据中心后端服务，设计数据集 CRUD、分片上传合并校验、用户积分结算等核心业务模块，通过 Guard + Interceptor 实现统一鉴权与日志链路追踪。  
- 设计分片上传服务端逻辑：接收分片并持久化至对象存储，基于 **Redis** 记录分片上传状态与过期清理策略，全部分片到齐后触发合并校验（MD5 一致性），支撑 **100GB** 级文件可靠上传。  
- 开发数据集交易与积分结算 API，基于 **PostgreSQL** 设计数据集元信息表、交易流水表与积分明细表，使用事务保证交易与积分变动的原子性。  
- 封装 WebSocket Gateway（NestJS `@WebSocketGateway`），实现行情数据的服务端推送、房间订阅管理与连接生命周期管控。
